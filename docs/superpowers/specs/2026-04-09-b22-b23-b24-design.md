## B-23 — Hide Simulation tab on desktop

**Root cause:** `AppTab::Simulation` tab button is rendered unconditionally in `render_app_tab_bar`. On desktop, the body shows "Simulation not available in this context." — a confusing dead tab.

**Fix:** Gate the `selectable_value` call for `▶ Simulation` with `#[cfg(target_arch = "wasm32")]`.

**File:** `crates/ferrite-egui/src/app_window.rs`

**Details:**
- The `AppTab::Simulation` enum variant is retained — it is referenced by `simulation_override` logic and the match arm in `render_full_window`.
- The match arm showing "Simulation not available…" becomes dead code on desktop but causes no compile error (it's still a valid arm). It can be left as-is or also gated; gating is cleaner.
- No changes needed to `AppTab` enum itself.

## B-24 — Desktop sprite editor regression

**Root cause:** `render_sprites_tab` (in `ferrite-egui`) sets `sprite_editor = None` when the selected key changes, with the comment "caller must recreate for the new key". The webapp's `WebApp::update()` handles this correctly (lines 94–105 of `crates/ferrite-webapp/src/app.rs`). The desktop caller in `src/tray/app_window.rs` was never updated with the equivalent logic.

**Fix:** Add a "wire sprite editor" block to the desktop `open_app_window` loop, immediately after `render_full_window`:

```rust
// Wire sprite editor: recreate when a key is selected but no editor is loaded
if s.sprite_editor.is_none() {
    if let Some(ref key) = s.selected_sprite_key.clone() {
        let config_dir = crate::config::config_path()
            .parent()
            .map(|p| p.to_path_buf())
            .unwrap_or_else(|| std::path::PathBuf::from("."));
        if let Ok(es) = load_editor_state_from_sheet(key) {
            let mut ed = make_desktop_sprite_editor(es, &config_dir);
            ed.is_builtin = key.starts_with("embedded://");
            s.sprite_editor = Some(ed);
        }
    }
}
```

**File:** `src/tray/app_window.rs`

**Details:**
- Mirrors the webapp pattern exactly.
- `load_editor_state_from_sheet` already handles both `embedded://` paths and filesystem paths.
- Placement: after `render_full_window`, before the existing `OpenEditorRequest` handler (so Edit… from Config tab overrides the auto-load if both fire on the same frame — unlikely but safe).

## B-22 — PNG import on webapp (session-only)

**Constraint:** Session-only for now. Persistence across reloads (localStorage for uploaded PNG bytes) is a future backlog item.

**Architecture:** Signal flag in shared state → webapp-side async file pick → register in-memory → auto-load in sprite editor.

### State additions

**`crates/ferrite-egui/src/app_window.rs` — `AppWindowState`:**
```rust
pub wants_png_import: bool,   // set by gallery panel; consumed by platform layer
```
Initialized to `false`.

**`crates/ferrite-webapp/src/app.rs` — `WebApp`:**
```rust
pending_png_bytes: Option<crossbeam_channel::Receiver<(String, Vec<u8>)>>,
```
`(filename, raw_png_bytes)` — filename is used as the storage key.

### Button (shared panel)

**`crates/ferrite-egui/src/app_window.rs` — `render_sprites_tab`:**

Below the gallery `ScrollArea`, add:
```rust
#[cfg(target_arch = "wasm32")]
if ui.button("Import PNG…").clicked() {
    s.wants_png_import = true;
}
```

### Async handler (webapp)

**`crates/ferrite-webapp/src/app.rs` — `WebApp::update()`:**

```rust
// Consume wants_png_import signal
if self.state.wants_png_import && self.pending_png_bytes.is_none() {
    self.state.wants_png_import = false;
    let (tx, rx) = crossbeam_channel::bounded(1);
    wasm_bindgen_futures::spawn_local(async move {
        if let Some(fh) = rfd::AsyncFileDialog::new()
            .add_filter("PNG", &["png"])
            .pick_file()
            .await
        {
            let name = fh.file_name();
            let bytes = fh.read().await;
            tx.send((name, bytes)).ok();
        }
    });
    self.pending_png_bytes = Some(rx);
}

// Poll result
if let Some(ref rx) = self.pending_png_bytes {
    if let Ok((filename, png_bytes)) = rx.try_recv() {
        self.pending_png_bytes = None;
        let key = format!("user://{filename}");
        let json_stub = make_single_frame_json_stub(&png_bytes);
        if let Ok(json_bytes) = json_stub {
            self.sheet_loader.register(key.clone(), json_bytes, png_bytes);
            self.state.gallery.push(ferrite_egui::gallery::GalleryEntry {
                key: key.clone(),
                display_name: filename,
            });
            self.state.selected_tab = AppTab::Sprites;
            self.state.selected_sprite_key = Some(key);
            self.state.sprite_editor = None; // auto-recreated next frame
        }
    }
}
```

### JSON stub generation

When a raw PNG is imported with no accompanying Aseprite JSON, we generate a minimal stub so `WebSheetLoader::load_sheet` succeeds:
- Single frame: `{ x: 0, y: 0, w: image_width, h: image_height }`
- No tags, no sm_mappings, no chromakey, baseline_offset = 0

`make_single_frame_json_stub(png_bytes: &[u8]) -> anyhow::Result<Vec<u8>>` decodes the PNG header to get dimensions, then serializes a minimal `SpriteSheet`-compatible JSON.

**File:** `crates/ferrite-webapp/src/app.rs` (inline helper) or a small helper in `web_storage.rs`.

### Key scheme

Imported sprites use the `user://` prefix (e.g. `user://cat.png`). This is distinct from `embedded://` (bundled) and filesystem paths (desktop). The `WebSheetLoader` handles it identically to any other registered key. `is_builtin` stays `false`.

### Duplicate handling

If the user imports a file with the same name twice in the same session, `register` overwrites the previous entry in `WebSheetLoader`. The gallery entry is pushed again — a minor duplicate display issue. Acceptable for session-only V1; de-duplication is part of the persistence story.
